## 一、项目背景与目的

在大模型快速发展的背景下，法律领域对智能问答与信息抽取提出了更高要求。面对海量法律法规文档，如何构建自主知识库并结合本地 LLM 进行精准问答，是当前的重要研究方向。

本项目以"北大法宝"等公共法律数据库为数据源，通过网页爬虫下载法律法规全文，构建本地法规知识库，并利用 RAG（检索增强生成）+ Agent 技术构建允许多轮交互、任务调度的智能问答系统。系统全程本地部署，无需联网，保障数据安全。

### 核心目标

1. 部署本地 LLM（Ollama + 国产模型）和 Embedding 模型
2. 编写爬虫，爬取 30 部核心法律全文
3. 文档清洗、分段、向量化，构建 FAISS 索引
4. 实现 RAG 问答系统（检索 + 生成）
5. 引入 LangGraph Agent 框架，实现多步骤任务调度
6. 提供 FastAPI 服务端 + Vue 3 前端界面
7. 构建 100+ 条标注测试数据集，完成系统性评测

### 2.2 数据流

```
用户提问 → 意图识别（闲聊/法律）
  ├─ 闲聊 → LLM 直接回复
  └─ 法律 → 查询改写 → FAISS 向量检索 → Reranker 精排
            → 相邻条文扩展 → 拼接 Prompt → LLM 生成
            → 答案校验（不合格则重试）→ 返回用户
```

### 2.3 部署架构

```

  ├─ pgvector (PostgreSQL + 向量扩展) — 可选
  ├─ FastAPI (uvicorn) — API 服务
  └─ Ollama (qwen2.5:7b + bge-m3 + bge-reranker-v2-m3) — 外部服务
```

## 三、技术选型

| 模块 | 选型 | 理由 |
|:---|:---|:---|
| **本地 LLM** | Ollama + qwen2.5:7b | 国产模型，中文理解优秀，Ollama 一键部署 |
| **Embedding** | bge-m3 (1024d) | 中英双语，M3 架构支持多语言，MIT 开源 |
| **Reranker** | bge-reranker-v2-m3 | 与 bge-m3 同家族，Cross-Encoder 精排，CUDA 加速 |
| **向量索引** | FAISS (IndexFlatIP) | 纯内积检索，3753 条文档精度优先 |
| **Agent 框架** | LangGraph 1.2 | 状态图编排，支持条件路由 + 重试循环 |
| **分词** | jieba | 中文分词，用于 BM25 关键词检索 |
| **Web 框架** | FastAPI 0.115 | 高性能异步，自动 OpenAPI/Swagger 文档 |
| **前端** | Vue 3 + Vite + Pinia | 响应式 SPA + SSE 流式渲染 |
| **认证** | JWT (python-jose) | 无状态 Token 认证 |
| **爬虫** | requests + BeautifulSoup4 | 轻量级，适合结构化页面 |
| **包管理** | uv | 极速依赖管理，替代 pip |
| **容器化** | Docker + docker compose | 一键部署 |

### 为什么不选 X 而选 Y

| 对比 | 选择 | 原因 |
|:---|:---|:---|
| FAISS vs Milvus | FAISS | 数据量 3753 条，无需分布式 |
| LangGraph vs AutoGen | LangGraph | 状态图更适合固定流程，调试方便 |
| bge-m3 vs text2vec | bge-m3 | 中文法律文本语义理解更强 |
| qwen2.5:7b vs LLaMA3 | qwen2.5 | 中文训练数据更丰富，法律术语更准确 |

## 四、知识库构建

### 4.1 数据来源

通过 `scripts/crawler.py` 爬取北大法宝网站，获取 30 部核心法律全文：

| 类别 | 法律 |
|:---|:---|
| 宪法类 (6部) | 宪法、宪法修正案、立法法、全国人大组织法、国务院组织法、监察法 |
| 刑事类 (1部) | 刑法 (2023修正) |
| 民事类 (1部) | 民法典 |
| 行政法类 (6部) | 行政处罚法、行政复议法、行政强制法、行政许可法、治安管理处罚法、公务员法 |
| 经济法类 (6部) | 公司法、证券法、企业破产法、合伙企业法、个人独资企业法、信托法 |
| 社会法类 (5部) | 劳动法、社会保险法、食品安全法、环境保护法、道路交通安全法 |
| 知识产权类 (4部) | 专利法、商标法、著作权法、反不正当竞争法 |
| 金融类 (1部) | 票据法 |

### 4.2 文本解析

`src/chunking/parser.py` 实现中文法律文本的层次解析：

```
LawDocument
  ├─ Part（编）→ Chapter（章）→ Section（节）→ Article（条）
  └─ 提取: 法条号、正文、中文数字 → 整数映射
```

关键功能：中文数字转整数（`_cn_to_int`），支持「一」→「一千二百三十四」范围。

### 4.3 文本切分配置

| 参数 | 值 | 说明 |
|:---|:---:|:---|
| min_chunk_chars | 50 | 短条文自动合并到相邻 chunk |
| max_chunk_chars | 1500 | 单 chunk 上限 |
| merge_short_articles | True | 合并短条文减少碎片 |
| add_chapter_summary | True | 为每章生成摘要 chunk |

### 4.4 向量化

- 模型：bge-m3，输出 1024 维向量
- 批次大小：8（Ollama API 限制）
- 生成 3753 条向量
- 存储：FAISS IndexFlatIP（内积相似度）+ bm25_corpus.pkl（BM25 语料）

## 五、RAG 检索增强生成

### 5.1 检索流程

```
用户查询
  → FAISS 向量检索 (Top 10 候选)
  → chunk_type 过滤 (排除 chapter_summary)
  → bge-reranker-v2-m3 精排 (Cross-Encoder)
  → 返回 Top 5
```

### 5.2 检索配置

| 参数 | 值 | 说明 |
|:---|:---:|:---|
| 检索模式 | 纯向量 (FAISS) | 评测证明 BM25 混合为负优化 |
| top_k | 5 | 检索返回条数 |
| Reranker | bge-reranker-v2-m3 | Cross-Encoder 精排 |
| recall_k | 10 | 粗排候选数 |
| 相邻扩展 | window=2 | 返回前后 ±2 条相邻条文 |

### 5.3 Prompt 设计

系统使用专用法律 Prompt 模板，核心要素：

1. **角色设定**：「你是一位精通中国法律的专业法律助手」
2. **5 条约束**：引用法律名称和条款号、基于条文不编造、不足时诚实说明、简洁清晰、附加免责声明
3. **Few-shot 示例**：包含治安处罚种类、酒驾处罚等格式示范
4. **免责声明**：「⚠️ 以上内容基于现行法律法规整理，仅供参考，不构成专业法律意见。如涉及具体法律事务，请咨询执业律师。」

## 六、Agent 任务调度

### 6.1 LangGraph 工作流

系统基于 LangGraph 构建 6 节点状态图：

```
                    ┌──────────────┐
                    │    intent     │  意图识别
                    │ (法律/闲聊)    │
                    └──┬────────┬──┘
               legal  │        │  casual
              ┌───────▼──┐  ┌──▼─────────┐
              │ rewrite  │  │casual_reply │  闲聊直接回复
              │ 查询改写  │  └────────────┘
              └─────┬────┘
              ┌─────▼──────┐
              │  retrieve  │  FAISS 向量检索
              └─────┬──────┘
              ┌─────▼──────┐
              │  generate  │  LLM 生成回答
              └─────┬──────┘
              ┌─────▼──────┐
              │  validate  │  答案校验
              └──┬──────┬──┘
           pass  │      │ fail (retry)
              ┌──▼──┐   └──→ generate
              │ END │
              └─────┘
```

### 6.2 节点说明

| 节点 | 功能 | 关键逻辑 |
|:---|:---|:---|
| `intent` | 意图识别 | 正则 + LLM 分类，区分法律问题与闲聊 |
| `rewrite` | 查询改写 | LLM 将口语化问题改写为标准法律查询 |
| `retrieve` | 向量检索 | FAISS + Reranker 精排 |
| `generate` | 生成回答 | 拼接检索结果 + Prompt → LLM 生成 |
| `validate` | 答案校验 | LLM 自检：是否引用法条、是否编造、是否完整 |
| `casual_reply` | 闲聊回复 | 不经过检索直接回复 |

## 八、检索评测

### 8.1 评测方法

使用 100 条标注查询，对比 6 组 BM25 权重配置（0.0 / 0.1 / 0.3 / 0.5 / 0.7 / 0.9），指标：Recall@k、MRR、NDCG@10。

### 8.2 评测结果

| BM25 权重 | Recall@1 | Recall@5 | MRR |
|:---|:---:|:---:|:---:|
| **0.0（纯向量）** | **0.5200** | **0.7300** | **0.6113** |
| 0.1 | 0.1900 | 0.3000 | 0.1926 |
| 0.3 | 0.1900 | 0.3500 | 0.2219 |
| 0.5 | 0.3900 | 0.5400 | 0.3722 |
| 0.7 | 0.4200 | 0.5800 | 0.4392 |
| 0.9 | 0.4400 | 0.5800 | 0.4320 |

### 8.3 核心结论

1. **纯向量检索最优**：bge-m3 中文语义理解能力强，Recall@5 达到 73%（移除章级摘要噪声后的诚实基线）
2. **BM25 为负优化**：jieba 中文分词 + BM25 关键词匹配在法律文本场景下显著降低召回（最高 Recall@5 仅 58%，比纯向量低 22%）
3. **BM25 权重趋势**：权重越高越接近纯向量，说明向量分量始终是主要贡献
4. **Reranker + 相邻扩展已启用**：实际服务中额外加上精排和条文扩展，端到端效果更优

## 九、检索噪声优化

### 9.1 问题发现

在填充测试数据集时，发现「正当防卫」查询召回了包含 33 条军人违反职责罪的章级摘要 chunk。该 chunk 将整章几十条条文打包为一个文本块，对各种查询都有虚高的向量相似度。

### 9.2 优化方案

| 优化 | 方法 | 效果 |
|:---|:---|:---|
| chunk_type 过滤 | 跳过 `chunk_type == "chapter_summary"` 的 chunk | 彻底消除 33 条无关条文打包返回 |
| 内容截断 | 单条 chunk ≤ 1500 字符 | 保护 Prompt 不被噪声吞满 |
| Reranker 精排 | Cross-Encoder 逐对打分 | 自然排出与 query 不相关的 chunk |

### 9.3 效果对比

| 查询 | 优化前 | 优化后 |
|:---|:---|:---|
| 正当防卫 | 5 条中 3 条无关（含 33 条军人违反职责罪） | 核心法条在前 2 位，章级噪声清零 |
| 故意伤害量刑 | — | 5/5 全中（刑法 234 + 关联条文 + 治安处罚） |

## 十二、已知局限与改进方向

### 12.1 局限

| 问题 | 影响 | 改进方向 |
|:---|:---|:---|
| CPU 推理慢 (107s/次) | 流式/多轮超时 | 换 GPU 或更小的模型 |
| 15 条查询全部未命中 | &lt;15% 的查询无结果 | 修正法条号映射 + chunk 粒度调整 |
| 含"零"的条文解析失败 | 部分法条号无法匹配 | 正则扩展字符集 |
| 无 session 管理 | 多轮对话依赖前端传 history | 增加 Redis/session_id |
| 知识库静态 | 法律修订后需重新爬取 | 增加增量更新机制 |

### 12.2 后续优化

1. **模型升级**：qwen2.5:7b → qwen2.5:14b，提升生成质量
2. **上下文窗口**：增加长上下文支持，减少截断
3. **评测自动化**：LLM Judge 自动对比 expected_answer vs model_output，输出准确率/幻觉率
4. **多模态扩展**：支持法律文书 OCR、判决书结构抽取
